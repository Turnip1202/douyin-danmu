# 抖音弹幕项目修复文档

## 一、问题概述

本项目是一个抖音直播弹幕监听工具，在使用过程中遇到了以下问题：

1. **依赖安装错误**：`execjs` 包无法安装
2. **模块缺失**：`google.protobuf` 模块找不到
3. **编码问题**：Windows 环境下 `UnicodeEncodeError: 'gbk' codec can't encode character`
4. **签名生成失败**：WebSocket 握手返回 400 Bad Request

---

## 二、修复过程详解

### 2.1 依赖安装问题

**问题现象**：
```bash
ERROR: Could not find a version that satisfies the requirement execjs
```

**原因分析**：
- `execjs` 不是正确的包名，正确的包名是 `PyExecJS`

**修复方案**：
```bash
# 错误命令
pip install execjs

# 正确命令
pip install PyExecJS
```

**更新 requirements.txt**：
```txt
requests
websocket-client
PyExecJS
loguru
protobuf
```

### 2.2 Protobuf 模块缺失

**问题现象**：
```bash

```

**原因分析**：
- `douyin_pb2.py` 使用了 Protocol Buffers，需要安装 `protobuf` 库

**修复方案**：
```bash

```

### 2.3 Windows 编码问题（核心修复）

**问题现象**：
```bash
UnicodeEncodeError: 'gbk' codec can't encode character '\u0e11' in position 483527
```

**原因分析**：
- Windows 默认编码为 GBK，而 `execjs` 调用 Node.js 时使用 UTF-8 编码
- 编码不匹配导致中文字符无法正确处理

**修复方案**：

修改 `main.py`，添加编码设置并改用 `subprocess` 调用 Node.js：

```python
import os
import sys

# 设置 UTF-8 编码
os.environ['PYTHONIOENCODING'] = 'utf-8'

# Windows 系统控制台编码设置
if sys.platform.startswith('win'):
    try:
        sys.stdout.reconfigure(encoding='utf-8')
        sys.stderr.reconfigure(encoding='utf-8')
    except:
        pass

# 使用 subprocess 替代 execjs
import subprocess
import json

# 生成临时 JS 文件执行签名
js_code = f"""
{full_js}
var result = get_sign('{sign}');
console.log(JSON.stringify(result));
"""

with open('_temp_sign.js', 'w', encoding='utf-8') as f:
    f.write(js_code)

result = subprocess.run(
    ['node', '_temp_sign.js'],
    capture_output=True,
    text=True,
    encoding='utf-8',
    env={**os.environ, 'NODE_OPTIONS': '--no-warnings'}
)
```

### 2.4 WebSocket 握手失败

**问题现象**：
```bash
Handshake status 400 Bad Request
```

**原因分析**：
1. **参数硬编码**：`sub_room_id` 和 `user_unique_id` 是硬编码值
2. **签名值被污染**：`sign.js` 中的 `console.log` 输出混入了签名结果

**修复方案**：

#### 步骤 1：动态获取房间信息

修改 `get_room_id` 函数为 `get_room_info`，提取更多参数：

```python
def get_room_info(url):
    # ... 原有代码 ...
    
    # 提取 room_id
    match_list = re.findall(r'"roomId\\":\\"(\d+)\\",', res.text)
    if not match_list:
        match_list = re.findall(r'"room_id":(\d+)', res.text)
    room_id = match_list[0]
    
    # 提取 sub_room_id
    sub_room_match = re.findall(r'"sub_room_id":(\d+)', res.text)
    sub_room_id = sub_room_match[0] if sub_room_match else room_id
    
    # 提取 user_unique_id
    user_unique_match = re.findall(r'"user_unique_id":"?(\d+)"?', res.text)
    if not user_unique_match:
        user_unique_match = re.findall(r'"webcast_user_id":"?(\d+)"?', res.text)
    user_unique_id = user_unique_match[0] if user_unique_match else "7621938952584791552"
    
    # 提取 ttwid
    ttwid = res.cookies.get_dict().get('ttwid', '')
    
    return room_id, sub_room_id, user_unique_id, ttwid
```

#### 步骤 2：正确提取签名

过滤掉调试输出，只提取有效的签名值：

```python
if result.returncode == 0:
    output = result.stdout.strip()
    # 签名可能在最后一行（过滤调试输出）
    lines = output.split('\n')
    signature = None
    for line in reversed(lines):
        line = line.strip()
        if line and not line.startswith('[') and not line.startswith('✅') and not line.startswith('❌'):
            signature = line
            break
    
    if signature is None:
        signature = lines[-1].strip() if lines else output
```

## 九、新增Bug记录

### 9.1 日志不输出到控制台

**问题现象**：
- 启动应用后，弹幕监听日志只输出到文件 `logs/app.log`，控制台没有任何日志输出

**原因分析**：
- loguru 配置只添加了文件输出处理器，没有配置控制台输出
- Flask 的 debug 模式会自动重载，导致日志配置被重置

**修复方案**：
```python
# app/utils/logger.py
def get_logger(name: str = __name__):
    global _logger_configured
    
    # 只配置一次，避免重复添加处理器
    if not _logger_configured:
        # 配置控制台输出
        logger.remove()
        logger.add(
            sys.stdout,
            level="INFO",
            format="{time:YYYY-MM-DD HH:mm:ss} | {level} | {module}:{line} - {message}"
        )
        
        # 配置文件输出
        logger.add(
            os.path.join(log_dir, 'app.log'),
            rotation="10 MB",
            retention="7 days",
            level="INFO",
            encoding="utf-8"
        )
        
        _logger_configured = True
    
    return logger
```

**修改文件**：`app/utils/logger.py`

### 9.2 日志重复记录

**问题现象**：
- 每条日志在文件中重复出现3次

**原因分析**：
- `get_logger()` 函数被多次调用，每次调用都会添加新的日志处理器
- 没有使用标志位控制配置只执行一次

**修复方案**：
- 添加全局标志 `_logger_configured`，确保日志配置只执行一次
- 在 `run.py` 中提前初始化日志配置

### 9.3 Windows控制台中文乱码

**问题现象**：
- 控制台输出的中文显示为乱码（如 `銆愮敤鎴?754458315097銆戝彂鍑哄脊骞曪細`）

**原因分析**：
- Windows 命令行默认使用 GBK 编码
- 日志输出使用 UTF-8 编码，导致编码不匹配

**修复方案**：
```python
def _setup_console_encoding():
    """设置控制台编码，解决Windows中文乱码问题"""
    if sys.platform.startswith('win'):
        try:
            import ctypes
            kernel32 = ctypes.windll.kernel32
            kernel32.SetConsoleOutputCP(65001)  # UTF-8
            kernel32.SetConsoleCP(65001)  # UTF-8
        except Exception:
            pass
```

### 9.5 删除房间时外键约束错误

**问题现象**：
```json
{
   "code": 500,
   "message": "(sqlite3.IntegrityError) NOT NULL constraint failed: danmus.room_id"
}
```

**原因分析**：
- 删除房间时没有先删除关联的弹幕记录
- SQLite 的外键约束导致删除失败

**修复方案**：
```python
def delete(self, room_id: str):
    """删除房间（同时删除关联的弹幕和点赞记录）"""
    room = self.get_by_room_id(room_id)
    if room:
        # 先删除关联的弹幕和点赞记录
        from app.models import Danmu, Like
        self.db.query(Danmu).filter(Danmu.room_id == room_id).delete()
        self.db.query(Like).filter(Like.room_id == room_id).delete()
        # 再删除房间
        self.db.delete(room)
        self.db.commit()
        return True
    return False
```

**修改文件**：`app/repositories/room_repo.py`

### 9.6 房间ID存储混乱

**问题现象**：
- 用户输入短房间ID（如 `459565028332`），但开始监听后系统获取的长ID被存储到数据库
- 导致弹幕数据关联错误

**原因分析**：
- 抖音直播URL中的ID（短ID）与实际API使用的ID（长ID）不同
- 需要同时存储短ID和长ID

**修复方案**：
```python
# app/models/__init__.py
class Room(Base):
    room_id = Column(String(64), unique=True, nullable=False, index=True)  # 用户输入的短ID
    long_room_id = Column(String(64))  # 抖音返回的长ID
    sub_room_id = Column(String(64))  # 子房间ID
    user_unique_id = Column(String(64))  # 用户唯一ID
    ttwid = Column(String(512))  # Cookie
```

**修改文件**：`app/models/__init__.py`
